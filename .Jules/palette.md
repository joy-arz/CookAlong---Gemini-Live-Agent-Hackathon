## 2024-03-20 - React Native Permission Screen Visibility & A11y
**Learning:** Permission screens in React Native with a dark background (`backgroundColor: '#000'`) can render text invisible if the default text color is dark. Also, when multiple permissions are required (e.g., Camera and Audio), providing a single button to request both makes it more intuitive for the user than failing silently or asking separately if one was skipped. Added explicit accessibility descriptions so screen readers provide more context than just "Grant Permission" or "Connect Server" / "Start Cooking".
**Action:** When creating permission/fallback screens, always explicitly define text colors matching the background theme. When requesting multiple permissions (Camera & Audio), group them in the same user action (button tap) and provide clear screen reader context via `accessibilityLabel`. Use `aria-live="polite"` on dynamically changing status texts to inform screen reader users of state changes.
